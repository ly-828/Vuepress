# vue

# 基本形式

vue通过一个文件引入，利用挂载的方式绑定dom元素

其基本形式可以表现为：

const app = new Vue({
el:'#app',
data:{
    message:0

},
methods:{
    add:function(){
        this.message++;
    },
    sub:function(){
        this.message--;
    }
}
})

在这里可以看到，vue是一个对象，通过new一个vue实例，我们在里面修改data等属性和methods属性（属性里面嵌套属性）

# mvvm与mvc

MVC是包括view视图层、controller控制层、model数据层。各部分之间的通信都是单向的。 view《——》control都作用于model

MVVM包括view视图层、model数据层、viewmodel层。各部分通信都是双向的。 MVC的分离不如MVVM。view和model有一定的依赖性

# 动态绑定 v-bind

  • v-bind 动态绑定 ,语法糖为 “：” 这是用来绑定 {}里面的东西

    不能使用 ，{{}}只能放在标签内容中 &&语法糖 img:src="url",

    动态绑定其中的属性，让其变成变量的形式

    v-bind也能绑定class属性,可以创建对象

      <div :class{classname：value_bool,classname:value_bool}></div>

通过布尔值设置class是否会被展示 也可以直接绑定style属性

    <div :class="{classname:bool}">
    <div :style="{stylename:value}"> 例如<div :style={fontSize:'50px',backgroud=changeback}>,
单引号需要加，否则在value端会被解析为data中的变量，这样会出问题。。

# 计算属性

计算属性computed在微信里面是没有的

computed:{
    Fullname：function(){
        return this.firstname+this.lastname;
    }
}

计算属性一般情况下不设置set方法

computed:{
    Fullname：{
        set:function(){
            通过set来读取值，大部分情况都不用
        }
        get：function(){
        return this.firstname+this.lastname;
        }
    }
}

这种方法在实现时可以不加小括号 Fullname 直接被视为函数

计算属性和方法有一些区别，计算属性在调用时会先执行，方法没有缓存，执行几次调用几次，降低速度 典型函数：fliter，reduce，map
